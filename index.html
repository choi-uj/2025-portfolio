KAIST 녹색성장지속가능대학원 웹사이트 클론 코딩

# 제작기간 2025. 06. 01 ~ 2025. 06. 16
# 기술 스택
  • HTML5
  • CSS3 ( Flexbox, Grid 포함 )
  • JavaScript ( 기초 DOM 조작, 간단한 인터랙션 )
# 개요

실제 존재하는 KAIST 녹색성장지속가능대학원(GSIS)의 공식 웹사이트를 클론 코딩하여, 웹 퍼블리싱 및 프론트엔드 개발 역량을 향상시키기 위한 개인 프로젝트입니다.

# 목표
  • 실제 웹사이트 구조 및 디자인을 분석하고 유사하게 구현
  • HTML, CSS, JavaScript를 활용한 반응형 웹 구현 연습
  • 웹 표준 및 접근성을 고려한 마크업 구성 연습
# 주요 구현내용
  • 메인 페이지 구조 및 서브 페이지 복제
  • 상단 메뉴바 및 드롭다운 메뉴 구현
  • 반응형 레이아웃 구현 (PC/태블릿/모바일 대응)
  • 간단한 슬라이더/탭 등 인터랙션 기능 추가
# 느낀 점 및 개선 사항
  • 실제 웹사이트를 모방하며 구조화된 HTML 작성 및 CSS 설계 경험을 쌓을 수 있었습니다.
  • 웹 접근성 및 유지보수성을 고려한 코딩의 중요성을 체감했습니다.
  • 향후에는 React 등의 프레임워크를 활용하여 컴포넌트 기반으로 리팩토링할 계획입니다.

Wolo Brothers 웹사이트 리디자인

# 제작기간 2025. 06. 17 ~ 2025. 07. 30
# 기술 스택
  • Figma ( 리디자인 )
  • HTML5, CSS3 ( Flexbox )
  • JavaScript ( 기초 DOM 조작, 간단한 인터랙션 )
  • React + vite ( Rebuild )
# 개요

원본 WoloBrothers 웹사이트는 콘텐츠가 매력적이지만 전반적인 디자인 일관성과 사용자 경험(UX) 측면에서 개선의 여지가 있어 리디자인을 기획했습니다.

# 목표
  • 직관적이고 현대적인 레이아웃으로 리디자인
  • 반응형 레이아웃 및 시멘틱 마크업 적용
  • 재사용 가능한 컴포넌트 구조로 확장성 확보
# 주요 구현내용
  • 리디자인 (HTML/CSS)
    • 기존 사이트의 구조를 분석 후 와이어프레임 및 스타일 가이드를 직접 제작
    • Flexbox와 Grid를 활용한 반응형 레이아웃 구성
    • 애니메이션(hover, scroll 효과)으로 인터랙션 강화
  • 리빌드 (React + Vite)
    • 컴포넌트 단위로 각 섹션 분리
    • Props를 통한 데이터 관리로 유지보수성 향상
    • GitHub Pages에 React 앱 자동 배포 설정
# 느낀 점 및 개선 사항
  • 단순한 HTML 리디자인에서 React 리빌드로 발전시키면서 SPA 구조와 컴포넌트 사고방식을 익힐 수 있었습니다.
  • 존 디자인을 재해석하면서 UI/UX 설계 능력과 프론트엔드 아키텍처 이해도가 향상되었습니다.
  • 향후에는 이미지 최적화와 코드 분할을 통해 로딩 속도를 개선할 예정입니다.

Seoul Gardening Club 웹사이트 리디자인

# 제작기간 2025. 08. 04 ~ 2025. 10. 02
# 기술 스택
  • Figma ( 리디자인 )
  • React + vite
  • CSS3 ( 공통 요소 ), SCSS ( 컴포넌트 )
  • MySQL, Postman, Node.js
  • Kakao chatbot
# 개요

기존의 서울가드닝클럽(Seoul Gardening Club) 웹사이트는 정원 제작 서비스라는 목표를 잘 담고 있지만, 전반적으로 디자인이 통일되지 않았고, 정보 전달 구조가 다소 복잡했습니다.

# 목표
  • Figma 기반의 UI/UX 리디자인을 통해 기존 사이트의 정보 구조와 비주얼 디자인을 개선
  • React + Vite 환경에서 재사용 가능한 컴포넌트 구조 구현
  • 카카오 챗봇 기능 연동으로 사용자 편의성 강화
  • MySQL 연동 및 API 통신 테스트(Postman)를 통한 백엔드 연동 구조 설계(Localhost)
# 주요 구현내용
  • React + Vite 프론트엔드
    • SCSS 모듈화를 통한 유지보수성 향상
    • Kakao Chatbot API 연동으로 사용자 편의 기능 추가
  • Node.js + MySQL 백엔드
    • Express.js 기반 REST API 서버 구현
    • CRUD 기능(데이터 생성, 조회, 수정, 삭제) 완성
    • MySQL 데이터베이스 모델링 및 쿼리 최적화
    • CORS 설정으로 프론트엔드와의 원활한 통신 확보
# 느낀 점 및 개선 사항
  • 프론트엔드와 백엔드를 모두 직접 구현하면서 데이터 흐름의 전반적인 구조를 이해할 수 있었습니다.
  • Kakao 챗봇을 연동하면서 실제 서비스형 기능 구현의 복잡성을 이해하게 되었습니다.
  • 단순한 리뷰 게시판이 아닌 회원가입, 로그인, 로그아웃 기능도 구현해 볼 예정입니다.

AI 이미지 제작 project

# 제작기간 2025. 09. 08 ~ 2025. 09. 23
# 기술 스택
  • ComfyUI
  • 한글2022 ( 결과보고서 작성 )
# 개요

ComfyUI를 활용하며, 프롬프트와 모델 변경에 따라 생성되는 결과물이 달라지는 점에 흥미를 느껴 프로젝트를 진행했습니다. 다양한 모델과 노드 배치를 시험해보고, 그에 따른 결과물을 비교·분석하는 것을 목표로 삼았습니다.

# 목표
  • 여러 모델을 사용하여 결과물의 퀄리티 비교
  • 저사양 컴퓨터에서도 원활하게 동작 가능한 모델 탐색
  • 프롬프트 변화를 통해 결과물의 차이를 비교·분석
# 느낀 점 및 개선 사항
  • 모델과 노드 배치에 따라 결과물의 특성이 크게 달라지는 것을 체험하며, AI 생성 과정의 민감성을 이해하게 되었습니다.
  • 저사양 환경에서도 효율적으로 사용할 수 있는 모델과 최적화 방법을 찾아보는 경험을 통해 실무 적용 가능성을 고민하게 되었습니다.
